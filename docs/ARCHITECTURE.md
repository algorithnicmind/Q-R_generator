## Component Breakdown

### 1. Frontend Layer

The frontend is built with **vanilla HTML, CSS, and JavaScript** for simplicity and portability.

| Component             | Responsibility                                         |
| --------------------- | ------------------------------------------------------ |
| **QR Generator Page** | Form to create new QR codes, select type, preview QR   |
| **Dashboard Page**    | List all QR codes, view stats, manage status           |
| **QR Stats Page**     | Detailed analytics for individual QR codes             |
| **Error Pages**       | Display messages for expired/disabled/invalid QR codes |

### 2. Backend Layer

Built with **Node.js and Express.js**, the backend handles:

#### Middleware Stack

```javascript
app.use(cors()); // Cross-origin requests
app.use(express.json()); // JSON body parsing
app.use(logger()); // Request logging
app.use(errorHandler); // Centralized error handling
```

#### Route Structure

| Route               | Method | Handler                              |
| ------------------- | ------ | ------------------------------------ |
| `/api/qr`           | POST   | Create new QR code                   |
| `/api/qr/:id`       | GET    | Get QR details                       |
| `/api/qr/:id`       | PATCH  | Update QR status/URL                 |
| `/api/qr/:id`       | DELETE | Delete QR code                       |
| `/api/qr/:id/stats` | GET    | Get scan statistics                  |
| `/q/:id`            | GET    | **Redirect endpoint** (core feature) |

### 3. Database Layer

**MongoDB** stores all application data with the following design principles:

- **Separation of Concerns:** QR codes and scan logs in separate collections
- **Indexing:** `qr_code_id` indexed for fast lookups
- **Referential Integrity:** Scan logs reference QR codes by `qr_code_id`

## Key Design Decisions

### 1. Indirect Redirection Pattern

```
QR Code â†’ yourdomain.com/q/{id} â†’ Backend â†’ Real Destination
```

This pattern enables all tracking and control features.

### 2. Stateless Backend

- No session storage required
- Each request contains all necessary information
- Enables horizontal scaling

### 3. Atomic Scan Logging

```javascript
// Pseudo-code
async function handleScan(qrCodeId) {
  const qr = await QRCode.findOne({ qr_code_id: qrCodeId });

  if (!qr || !qr.is_active || isExpired(qr)) {
    return showErrorPage(reason);
  }

  // Log BEFORE redirect to ensure data capture
  await ScanLog.create({ qr_code_id: qrCodeId });

  return redirect(qr.target_url);
}
```

### 4. Minimal Data Collection

- Only timestamp logged
- No personal data (IP, device, etc.)
- Privacy-first approach
